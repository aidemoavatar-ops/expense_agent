# Agent Spec — ambient-expense-agent

## Overview

An event-driven ambient expense agent that acts as an automated triage queue for corporate expense
reports. It consumes incoming expense submissions (simulated as Pub/Sub messages) and routes them
based on transaction value — without any interactive user session.

**Routing logic:**
- **Under $100** — Auto-approved instantly by deterministic Python code (no LLM call).
- **$100 or more** — Routed through a pre-LLM safety screen, analyzed for compliance risk by a
  Gemini LLM, then paused for human review.

## Example Use Cases

| Scenario | Input | Expected Output |
|----------|-------|-----------------|
| Coffee receipt | `{amount: 8.50, category: "meals", employee: "alice"}` | Auto-approved, logged |
| Office supplies | `{amount: 42.00, category: "supplies", employee: "bob"}` | Auto-approved, logged |
| Laptop purchase | `{amount: 1200.00, category: "hardware", employee: "carol"}` | LLM risk analysis → flagged for human review |
| Flight booking | `{amount: 850.00, category: "travel", employee: "dave"}` | LLM risk analysis → flagged for human review |
| Borderline item | `{amount: 99.99, category: "meals", employee: "eve"}` | Auto-approved (under $100) |

## Tools Required

| Tool | Purpose | Auth |
|------|---------|------|
| Pub/Sub consumer (simulated) | Receive incoming expense report events | None (local simulation) |
| `auto_approve_expense` | Deterministic approval for low-value items (<$100) | None |
| `flag_for_human_review` | Write high-value items to a review queue/log | None |
| Pre-LLM safety screen | Validate expense payload before LLM call (block malformed/malicious input) | None |

No external SaaS APIs required for the prototype.

## Constraints & Safety Rules

- **NEVER auto-approve any expense ≥ $100** — must always go through LLM + human review.
- **NEVER delete or modify** the original expense submission record.
- **Pre-LLM screen must run** before the Gemini model sees any high-value expense payload.
- The agent must be **stateless per message** — no cross-message memory.
- All decisions must be **logged** (amount, category, employee, decision, timestamp).

## Success Criteria

- All expenses < $100 are auto-approved without an LLM call (verifiable via logs).
- All expenses ≥ $100 trigger an LLM compliance analysis and land in the human review queue.
- Pre-LLM screen rejects malformed payloads before they reach the model.
- Agent processes a batch of 10 simulated Pub/Sub messages correctly.

## Architecture Notes

- **Trigger**: Simulated Pub/Sub push (HTTP POST to a FastAPI endpoint in the prototype).
- **Low-value path**: Python function, no ADK agent involved.
- **High-value path**: ADK `Agent` with a compliance-review instruction + tools for flagging.
- **Deployment**: Prototype only — `--prototype` flag, no CI/CD or Terraform for now.
